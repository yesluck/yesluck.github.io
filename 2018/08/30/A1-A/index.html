Dynamic Programming (从Climbing Stairs开始的基本动态规划)

发表于 2018-08-30 | 分类于 Algorithms |

70. Climbing Stairs

动态规划还有一类有趣的题，便是将较大的问题分解为较小问题。例如这题：

You are climbing a stair case. It takes _n_ steps to reach to the top.

Each time you can either climb 1 or 2 steps. In how many distinct ways can you climb to the top?

Note: Given _n_ will be a positive integer.

这题的思路便是最基本的拆分。例如，我的台阶有10级，那么跳到第10级的方法数其实便是跳到第8级后再跳2级和跳到第9级后再跳1级的方法数之和。依此类推，递推公式为：

res[i] = res[i - 1] + res[i - 2];

那么，一步步向前递推，只需要人为为res[0]和res[1]赋值，那么整个递推链便成功完成。
本题核心代码为递推公式，完整代码如下所示：

class Solution {
    public int climbStairs(int n) {
        int[] res = new int [n];
        if(n == 0)  return 0;
        if(n == 1)  return 1;
        if(n == 2)  return 2;
        res[0] = 1;
        res[1] = 2;
        for(int i = 2; i < n; i++)
        {
            res[i] = res[i - 1] + res[i - 2];
        }
        return res[n - 1];
       }
}

变式题——746. Min Cost Climbing Stairs

On a staircase, the i-th step has some non-negative cost cost[i]assigned (0 indexed).

Once you pay the cost, you can either climb one or two steps. You need to find minimum cost to reach the top of the floor, and you can either start from the step with index 0, or the step with index 1.

在这题里，每一步的爬升增加了代价，我们需要求取爬到顶端的最小代价。

同样对问题进行拆分，我们需要逆推求取到达每一级所需要的最小代价。那么递推公式便如下所示：

localSumCost[i] = Math.min(localSumCost[i-1] + cost[i-1], localSumCost[i-2] + cost[i-2]);

而递推的起点便是localSumCost[0]和localSumCost[1]，按照题意它们都是0。于是完整代码如下所示。

class Solution {
    public int minCostClimbingStairs(int[] cost) {
        int[] localSumCost = new int [cost.length + 1];
        localSumCost[0] = 0;
        localSumCost[1] = 0;
        for(int i = 2; i < cost.length + 1; i++)
        {
            localSumCost[i] = Math.min(localSumCost[i-1] + cost[i-1], localSumCost[i-2] + cost[i-2]);
        }
        return localSumCost[cost.length];
    }
}

需要注意的是开设的数组长度是cost.length + 1，因为需要考虑第0级台阶，因此总的台阶数需要+1。

62. Unique Paths

啊，变成二维了，一次一步，同样的道理。简单一句，dp[i][j] = dp[i-1][j] + dp[i][j-1]。

63. Unique Paths II

在这题中，原始的二维空间里多了一个障碍物。为了简化代码，我们先对上题的空间进行一个优化：对于起点到点(i,j)的路径总数：ways[j]= 起点到点(i-1, j)的路径总数：ways[j] + 起点到点(i, j-1)的路径总数 ways[j-1]，于是我们就得到递推式：ways[j] = ways[j] + ways[j-1]。这样依次看每一行，每行结果累积，就不需要设置二维矩阵了。

那么本题循环中的核心部分便成为：

if(obstacleGrid[i][j] == 1) dp[j] = 0;
else    if(j > 0)   dp[j] = dp[j] + dp[j - 1];

因此，其实这题和上题都可以化成一维情况。

120. Triangle

Given a triangle, find the minimum path sum from top to bottom. Each step you may move to adjacent numbers on the row below.

看起来是三角，其实还是可以化成一维的题。因为这题从下到上，每次都只需要看一层的情况，还不需要回头，所以把结果直接累计一波上来就行。

设置一个一维数组dp，为了减少初始化的代码量可以将dp长度多开一个（相当于给三角形的底端多加了一层全0）。于是核心公式便是：

dp[j] = Math.min(dp[j], dp[j+1]) + triangle.get(i).get(j);

意思便是，在这一层，第j个元素的累积和等于下一层第j或第j+1个元素的累积和+本层第j个元素。具体选哪一个，就按照题目要求，挑最小的那个就行。

变式题——看上去不那么直接

279. Perfect Squares

Given a positive integer _n_, find the least number of perfect square numbers (for example, 1, 4, 9, 16, ...) which sum to _n_.

嗯，求组成某个数的最少完全平方数个数。先假设每个数只能由1组成，再从小到大依次来看，试着减去可减的平方数，来找到最为科学的组合：

num[i] = Math.min(num[i], num[i - j * j] + 1);

322. Coin Change

You are given coins of different denominations and a total amount of money _amount_. Write a function to compute the fewest number of coins that you need to make up that amount. If that amount of money cannot be made up by any combination of the coins, return -1.

这题跟上题（279. Perfect Squares）真的很相似了……因为核心代码同样是：

dp[i] = Math.min(dp[i], dp[i - coins[j]] + 1);

但是还是有些不同的。主要的不同在于coins数组中的排列顺序并不是按照数字大小顺序排列的，因此还需要进行一波操作：首先，对于dp数组的初始化，因为最终需要求最小值，所以初始的数据应该是越大越好，在java中可以设置为Integer.MAX_VALUE；其次，对于核心代码的更新，是需要设置条件的：需要金额起码得达到这枚硬币的面值、dp[i - coins[j]]不是Integer.MAX_VALUE（不然这波更新就毫无意义了，还会溢出到负数hhh），归纳成代码便是：

if(i - coins[j] >= 0 && dp[i - coins[j]] != Integer.MAX_VALUE)

于是整体代码便是：

class Solution {
    public int coinChange(int[] coins, int amount) {
        int[] dp = new int [amount + 1];
        for(int i = 1; i <= amount; i++)
        {
            dp[i] = Integer.MAX_VALUE;
            for(int j = 0; j < coins.length; j++)
            {
                if(i - coins[j] >= 0 && dp[i - coins[j]] != Integer.MAX_VALUE)
                    dp[i] = Math.min(dp[i], dp[i - coins[j]] + 1);
            }
        }
        if(dp[amount] == Integer.MAX_VALUE || dp[amount] < 0)  return -1;
        return dp[amount];
    }
}

139. Word Break

Given a non-empty string _s_ and a dictionary _wordDict_
containing a list of non-empty words, determine if _s_ can be segmented into a space-separated sequence of one or more dictionary words.

Note:

  • The same word in the dictionary may be reused multiple times in the segmentation.
  • You may assume the dictionary does not contain duplicate words.

这题的主要目标是看字符串是否能够被拆分为词典中的词的组合。按照动态规划的思路，我们将从左到右按字符依次查看。设置一个一维数组dp，为了减小代码量跟之前一样把长度多开一个（dp[0]记为true）。

看的顺序大概是这样：先以第1个字母为词语终点，即只看第1个字母，如果在词典里就记为true，否则记为false；再以第2个字母为词语终点，即看第1~2、第2个字母分别组成的两个词……【其中，第1~2个字母组成的词如果在词典里就记为true，否则记为false；而第2个字母组成的词要记为true需要什么条件呢？那就是它之前的true了，它自己也要在词典里，具体公式如下所示：】

if(dp[j] == true && wordDict.contains(tmp)) dp[i] = true;

其中i便是外循环，即词语终点；j为内循环，即词语起点。

256. Paint House

There are a row of n houses, each house can be painted with one of the three colors: red, blue or green. The cost of painting each house with a certain color is different. You have to paint all the houses such that no two adjacent houses have the same color.

The cost of painting each house with a certain color is represented by a n x 3 cost matrix. For example,costs[0][0] is the cost of painting house 0 with color red; costs[1][2] is the cost of painting house 1 with color green, and so on… Find the minimum cost to paint all houses.

Note: All costs are positive integers.

这是一个二维的动态规划问题（真的来到二维了）。第一维是房子编号、第二维是颜色编号。每个房子的染色代价可以这样计算：如果当前房子染红色的话，最小代价将是上一个房子的绿色和蓝色的最小代价+当前房子染红色的代价，对另外两种颜色也是如此。因此核心代码便是：

dp[i][0] = min(dp[i-1][1], dp[i-1][2]) + costs[i][0];
dp[i][1] = min(dp[i-1][0], dp[i-1][2]) + costs[i][1];
dp[i][2] = min(dp[i-1][0], dp[i-1][1]) + costs[i][2];

变式题

64. Minimum Path Sum

做完前面的题，这题就太简单了，一遍就能写对。。。

复习补充：以下代码称为“我也会写骚代码”。

class Solution {
    public int minPathSum(int[][] grid) {
        for(int i = 0; i < grid.length; i++)
            for(int j = 0; j < grid[0].length; j++)
                grid[i][j] += (i == 0 && j == 0 ? 0 :Math.min((i == 0 ? Integer.MAX_VALUE : grid[i-1][j]), (j == 0 ? Integer.MAX_VALUE : grid[i][j-1])));
        return grid[grid.length-1][grid[0].length - 1];
    }
}

也许骚代码的一个特征就是尽量少用大括号尽量少行数吧（然而还是两层循环啊喂！）……
