翻转部分

206. Reverse Linked List

这题要求反转链表。思路大概有两种。经过实测，第一种代码长速度快、第二种代码段速度慢。

第一种方法：迭代实现。初始化时先设置好当前节点p和新的链表头newHead。随后进入迭代：

  1. 设置临时变量temp = p.next（以防当前p.next之后没有索引）
  2. 进行方向调换，即将p.next指向newHead。

  3. 向右移动，准备进行下一轮的变换：

    newHead = p;
p = temp;

完整代码如下所示：

class Solution {
    public ListNode reverseList(ListNode head) {
        if(head == null || head.next == null)   return head;
        ListNode p = head;
        ListNode newHead = null;
        while(p != null)
        {
            ListNode temp = p.next;
            p.next = newHead;
            newHead = p;
            p = temp;
        }
        return newHead;
    }
}

这题虽然是个easy题，但是第二种方法（递归实现）还是比较折磨人的。主要步骤是以下几步：

第一步，将指针迅速移到链表最后（倒数1、2位分别为newHead、head）。实现这一步实际上是通过递归的首步操作进行的，即如下代码片段：

class Solution {
    public ListNode reverseList(ListNode head) {
        if(head == null || head.next == null)   return head;
        ListNode newHead = reverseList(head.next);
        head.next.next = head;
        head.next = null;
        return newHead;
    }
}

第二步，进行位置调换。这里需要进行两步操作：

head.next.next = head;
head.next = null;

那么head.next = null;这步是否会影响下一层递归中head.next.next = head;的赋值过程呢？这个问题引用网上的博客分析如下：

我们在每次递归中，处理的对象都是 head->next 这个对象，那么这个对象在这层递归过程中，就是一个局部变量而已，我们设置 head->next->next 的指向，并没有修改 head->next 的值哦，而是通过我们存储的局部变量 head->next 来寻找到我们需要改变的对象而已。

所以说，因为每层递归之中的 head->next 的值都是相互不会影响的，所以 head->next = nullptr 的运行结果，就是我们期望看到的，整个链表的第一个节点的 next 指针被设置为空，此时返回正好达成题意。

92. Reverse Linked List II

这题要求翻转的是链表中其中一部分，因此在上题的基础上有所扩展，需要增加cur和newEnd变量。其中cur变量表示待翻转部分的前一个节点，newEnd变量代表待翻转部分的第一个节点（也是这部分翻转后的新终点）。

之后的翻转步骤和上一题可谓是完全一样。翻转结束后分别将newEnd接到后半部分（原本是接到null）、将cur接到newHead上即可。

代码如下所示：

class Solution {
    public ListNode reverseBetween(ListNode head, int m, int n) {
        ListNode dummy = new ListNode(0);
        dummy.next = head;
        ListNode cur = dummy, newHead = null;
        for(int i = 1; i < m; i++)  cur = cur.next;
        ListNode newEnd = cur.next, p = newEnd, temp = null;
        for(int i = m; i <= n; i++)
        {
            temp = p.next;
            p.next = newHead;
            newHead = p;
            p = temp;
        }
        cur.next.next = p;
        cur.next = newHead;
        return dummy.next;
    }
}

24. Swap Nodes in Pairs

这题属于第一题的变式题，需要两个两个地分别交换相邻的两个节点。同样也可以分别使用迭代和递归的方法进行解答。

第一种方法依旧是迭代。不过因为这次的每波操作涉及到两个节点再附带上之前那个节点，所以在初始化过程中要先在链表的最前面加上一个辅助节点，随后便能进入迭代。

在每轮迭代中，假设需要交换的两个节点分别是左节点和右节点，左节点之前的叫做先辈节点吧。在每次迭代开始时，我们只知道先辈节点和左节点，所以要首先赋值右节点。随后按顺序进行以下操作：

  1. 左节点不指向右节点，指向右右节点（篡夺右节点的地位第一步）
  2. 先辈节点不指向左节点，指向右节点（先辈钦定，把右节点往左拉）
  3. 右节点不指向右右节点，指向左节点（右节点无奈让位，彻底归左）

最后，为了能够使得迭代过程得以继续，需要为下一轮的迭代指定左节点和先辈节点。此时左节点右移后变成了下一轮的先辈节点，右右节点其实就是下一轮的左节点。这样的思路写下来如下所示：

class Solution {
    public ListNode swapPairs(ListNode head) {
        if(head == null || head.next == null)   return head;
        ListNode dummy = new ListNode(0);
        dummy.next = head;
        ListNode senpai = dummy;
        ListNode left = head;
        ListNode right;
        while(left != null && left.next != null)
        {
            right = left.next;
            left.next = right.next;
            senpai.next = right;
            right.next = left;
            senpai = left;
            left = left.next;
        }
        return dummy.next;
    }
}

第二种方法是递归遍历到链表末尾，然后先交换末尾两个，然后依次往前交换。代码非常简洁，但理解难度还是有一些的。

同样也是先把指针移动到最后，再从后向前进行交换。代码如下所示（最好画画图比较明白）：

class Solution {
    public ListNode swapPairs(ListNode head) {
        if(head == null || head.next == null)   return head;
        ListNode t = head.next;
        head.next = swapPairs(head.next.next);
        t.next = head;
        return t;
    }
}

328. Odd Even Linked List

这题需要将奇偶节点分类排序。自己想了一种比较直接的解法，需要事先记录奇数头节点（其实就是head）、奇数尾节点、偶数头节点、偶数尾节点，其中为了找到尾节点需要先遍历一遍链表。随后依次进行p.next = p.next.next;操作直到最后。最后将奇数尾节点和偶数头节点相连、偶数尾节点指向null即可。该方法比最快的方法仅慢了1ms，应该还算是比较直截了当的。

成环部分

141. Linked List Cycle

确定链表里是否有环。这题是个很经典的快慢指针思路的题——一个快指针一次跑两格、一个慢指针一次跑一格。如果有成环那么它们总有一天会相遇。有了这个思路之后便十分简单了。

public class Solution {
    public boolean hasCycle(ListNode head) {
        if(head == null || head.next == null)    return false;
        ListNode slow = head;
        ListNode fast = head;
        while(slow.next != null && fast.next != null && fast.next.next != null)
        {
            slow = slow.next;
            fast = fast.next.next;
            if(slow == fast)    return true;
        }
        return false;
    }
}

这题属于紧接上题背景之后的一道题。需要找到相交点入口。这里需要再将慢指针置于链表头、快指针原地不动，二者重新开始以步数为1同时出发。（需要计算证明）

237. Delete Node in a Linked List

并不需要解释。

class Solution {
    public void deleteNode(ListNode node) {
        node.val = node.next.val;
        node.next = node.next.next;
    }
}

19. Remove Nth Node From End of List

快慢指针思路，看到链表里的“Nth Node”就应该想到快慢指针。快指针先走N步，随后快慢指针一起往终点走，此时慢指针的指向便是“Nth Node”。然后remove过程就一行就行了。

82. Remove Duplicates from Sorted List II

这题也是得先放一个dummy，以防head的修改。设置pre指针先指向dummy，cur指针先指向head。

然后核心步骤主要有以下几个：

  1. 当cur的值和cur后一个节点的值相同时，疯狂将cur右移。这样就能将cur移到这一大堆重复数字的最后一个。为了实现这个操作需要使用while循环。
  2. 判断这波操作中是否存在重复的数字，其实就是判断cur是否还能够乖乖待在原地，即是否还是pre的后继。如果乖乖呆在原地，那么把pre和cur分别推进一格就行；如果cur已经飞奔到了千里之外，那么就需要使用pre.next = cur.next;操作使得跳过中间的重复部分，最后还需要将cur向右移动一格。

中间部分代码如下所示：

while(cur != null)
{
    while(cur.next != null && cur.val == cur.next.val)  cur = cur.next;
    if(pre.next == cur) pre = cur;
    else    pre.next = cur.next;
    cur = cur.next;
}

加减计算部分

369. Plus One Linked List

这题方法非常多样，大概比较直接的方法有两种，然后也有一种递归法比较简洁（但是递归感觉都比较不好一下子理解，所以先搁在这）。

第一种方法便是翻转、相加、再翻转。这个方法需要调用翻转函数两次，因此看起来是比较费时的。

第二种方法是比较巧妙的“找9法”，即找到右第一个非“9”数字，将其+1，并将后面的数字置0。当然也有一种情况是“全是9”，这种需要补一位就可以单独拿出来讨论了。

第二种方法自己写的代码如下所示：

class Solution {
    public ListNode plusOne(ListNode head) {
        if(head == null)    return head;
        ListNode dummy = new ListNode(1);
        ListNode right = null, p = head;
        while(p != null)
        {
            if(p.val != 9)  right = p;
            p = p.next;
        }
        if(right != null)
        {
            right.val += 1;
            while(right.next != null)
            {
                right = right.next;
                right.val = 0;
            }
            return head;
        }
        else
        {
            ListNode q = head;
            while(q != null)
            {
                q.val = 0;
                q = q.next;
            }
            dummy.next = head;
            return dummy;
        }
    }
}

2. Add Two Numbers

这题的思路十分直接，而且因为链表和数字本身就是倒序关系，因此并不存在需要翻转两次这样的操作，直接上去干就行。

class Solution {
    public ListNode addTwoNumbers(ListNode l1, ListNode l2) {
        int sum = 0;
        ListNode dummy = new ListNode(0), res = dummy;
        ListNode c1 = l1, c2 = l2;
        while(c1 != null || c2 != null)
        {
            if(c1 != null)  sum += c1.val;
            if(c2 != null)  sum += c2.val;
            res.next = new ListNode(sum % 10);
            res = res.next;
            if(c1 != null) c1 = c1.next;
            if(c2 != null) c2 = c2.next;
            sum /= 10;
        }
        if(sum == 1)    res.next = new ListNode(1);
        return dummy.next;
    }
}

其它部分

160. Intersection of Two Linked Lists

这题其实主要就是思路问题。要寻找两个链表A和B的交点，一个巧妙的思路就是把B和A分别接到A和B的后面，然后看到第一个重合的节点就是交点。

为了进行这波操作，需要将listA和listB指针同时向右移动，发现相等便返回结束程序。那么如何把B和A分别接到A和B的后面呢？这时当我们发现listA和listB中有一个是null时，便把它们接上去即可。

程序代码如下：

public class Solution {
    public ListNode getIntersectionNode(ListNode headA, ListNode headB) {
        ListNode listA = headA, listB = headB;
        while(listA != null && listB != null)
        {
            if(listA == listB)  return listA;
            listA = listA.next;
            listB = listB.next;
            if(listA == listB)  return listA;
            if(listA == null)   listA = headB;
            if(listB == null)   listB = headA;
        }
        return null;
    }
}

21. Merge Two Sorted Lists

这题一个巧妙的方法就是递归。递归依旧不好用文字解释，但是对着代码画图就会思路比较清楚了。

class Solution {
    public ListNode mergeTwoLists(ListNode l1, ListNode l2) {
        if(l1 == null)  return l2;
        if(l2 == null)  return l1;
        if(l1.val < l2.val) 
        {
            l1.next = mergeTwoLists(l1.next, l2);
            return l1;
        }
        else
        {
            l2.next = mergeTwoLists(l1, l2.next);
            return l2;
        }
    }
}

提高综合部分

234. Palindrome Linked List

Given a singly linked list, determine if it is a palindrome.

这题其实是之前很多题的综合，几个思路：1. 快慢指针找中点、2. 中点后链表部分翻转、3. 依次比对中点前后的链表值是否相等。

其中1. 快慢指针找中点的代码如下所示：

if(head == null || head.next == null)   return true;
ListNode slow = head, fast = head;
while(fast.next != null && fast.next.next != null)
{
    slow = slow.next;
    fast = fast.next.next;
}

其中2. 中点后链表部分翻转的代码如下所示：

ListNode p = slow.next, pre = head, newHead = null, temp = null;
while(p != null)
{
    temp = p.next;
    p.next = newHead;
    newHead = p;
    p = temp;
}
slow.next = newHead;

其中3. 依次比对中点前后的链表值是否相等的代码如下所示：

while(newHead != null)
{
    if(pre.val != newHead.val)  return false;
    pre = pre.next;
    newHead = newHead.next;
}
return true;

143. Reorder List

三个部分结合：快慢指针找中点、后半部分翻转、依次连接。其中依次连接部分如下所示：

ListNode first = head, second = newHead, temp1 = null, temp2 = null;
while(second != null)
{
    temp1 = first.next;
    first.next = second;
    temp2 = second.next;
    second.next = temp1;
    first = temp1;
    second = temp2;
}
